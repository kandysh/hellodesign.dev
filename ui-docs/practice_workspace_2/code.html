EASY

Design a simplified version of Twitter where users can post tweets, follow other users, and view a feed of tweets from users they follow.

Functional

  • check_circle Post a tweet (text only).
  • check_circle Follow/Unfollow users.
  • check_circle View user timeline (own tweets).
  • check_circle View home timeline (feed).

Non-Functional

  • speed High availability is critical.
  • timeline Read-heavy system (100:1 read/write ratio).
  • timer Fast feed generation (under 200ms).

System Architecture: Twitter Clone

1. Data Model

We will need primarily three tables to start: User, Tweet, and Follower.

Table Tweet {
  id: uuid PK
  user_id: uuid FK
  content: varchar(140)
  created_at: timestamp
}

2. High-Level Design

Clients connect through a Load Balancer to the API Gateway. Read requests (viewing timelines) are routed to a specialized Read Service backed by Redis caches.
