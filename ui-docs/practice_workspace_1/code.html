edit_note Architecture Notes

Proposed Solution: Fixed Window Counters

For our initial implementation, we will use a Fixed Window counter approach backed by Redis. It offers a good balance of simplicity and performance.

Data Model (Redis)

KEY: rate_limit:{api_key}:{timestamp_minute}
VALUE: Integer (incremented on each req)
TTL: 60 seconds

Pros & Cons

  • Pro: Memory efficient, only stores one integer per user per minute.
  • Pro: Extremely fast reads/writes using Redis INCR command.
  • Con: Spike traffic at window boundaries (e.g., 200 reqs at 12:00:59 and 200 at 12:01:01 allows 400 reqs in 2 seconds).
