Optimize Redis Eviction Policy

Performance Bottleneck Risk

The current cache configuration defaults to no eviction. For a high-throughput environment, this will lead to OOM errors. Update the policy to LRU to ensure volatile data cycles correctly.

maxmemory 2gb

Implement Exponential Backoff

Reliability Enhancement

The retry logic for the payment gateway webhook is currently linear. Implementing jitter and exponential backoff will prevent thundering herd scenarios during downstream outages.

webhook_worker.go 
// Current implementation
time.Sleep(time.Duration(retryCount) * time.Second)

// Recommended
backoff := math.Pow(2, float64(retryCount))
jitter := rand.Float64() * 1000
time.Sleep(time.Duration(backoff*1000 + jitter) * time.Millisecond)
