Architecture Notes

URL Shortener Design

1. API Design

RESTful API approach is preferred for simplicity.

POST api/v1/data/shorten - request: { longUrl: string } - response: { shortUrl: string }

2. URL Generation Strategy

Using a base62 encoding approach ensures short and URL-safe strings. We need a unique ID generator to prevent collisions.

  • Option A: Hash function + collision resolution. (Complex)
  • Option B (Chosen): Distributed Unique ID Generator (e.g., Snowflake) -> Base62 Encode.

3. Data Model

Since the data structure is simple and read-heavy, a NoSQL Key-Value store like DynamoDB or Cassandra is highly suitable.

Table: url_mapping PK: short_hash (String) - original_url (String) - created_at (Timestamp)
